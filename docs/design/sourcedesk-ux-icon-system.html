Strategic sourcing command center, styled for dense project work.

This mockup keeps the existing SourceDesk architecture, then normalizes the visual system: calmer navigation, stronger active states, consistent line icons, and status color only where users need to make a decision.

UX direction

Use icons as recognition aids, not replacements for important labels. Dense toolbars can use icon-only buttons with titles.
Reserve color for system state: saved, warning, destructive, generation, and provider/connector readiness.
Make the extension action-first: page extraction, CSV load, batch answering, visibility, and publish actions stay one thumb-scan apart.
Keep dangerous data operations visually grouped and separated from everyday project commands.

Application Mockup

Directly based on the existing app shell: topbar, sidebar projects, chat, context panel, supplier questions, working docs, and connector commands.

Chrome Extension Mockup

Action icons for the BidNet helper should stay compact, but never cryptic. Primary flow starts with extraction, moves to CSV load, then batch answer/publish.

Extension UX priorities

Use question-specific verbs: Extract, Answer, Comment, Publish, Private, Public.
Batch operations get filled icons and gold emphasis; one-off row actions stay quieter.
File outputs use distinct document shapes so JSON and CSV are not both generic download icons.
Visibility changes should show lock/globe semantics before any destructive publish action.

UX Implementation Notes

Small styling changes with outsized usability impact for the existing SourceDesk codebase.

Unify icon calls

Replace emoji/plain arrows with one helper that emits SVG by semantic id. Keep existing inline SVG until each view is touched.

Stable side nav

Group module links under Workspace, keep Projects and Chats separate, and use counts only for actionable items.

Clear risk states

Context limit warning, reached, private/public, and clear data need persistent color and icon patterns.

Editor toolbar

Use compact icon-only rich editor controls with titles; labels like H1/H2/H3 are acceptable because they are the standard glyph.
